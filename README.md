# SimplePage - Услуги

## Как запустить

1. Требования:
   - PHP 7.4 или выше
   - Веб-сервер (Apache/Nginx) или встроенный PHP сервер

2. Установка:
   - Скачать или клонировать репозиторий
   - Скопировать проект в директорию веб-сервера

3. Запуск:
   - Вариант 1: Использовать OSPanel (OSPanel) - скопировать проект в папку `OSPanel/home/`
   - Вариант 2: Использовать встроенный PHP сервер:
     ```bash
     php -S localhost:8000
     ```
   - Вариант 3: Использовать встроенный сервер WebStorm


## Что успел сделать

- Сделал Mock данные (эмуляция данных из БД)
- Сделал Модель с классом Service

- Сделал две страницы: со всеми услугами и детальной страницей услуги
- Разделил на отдельные файлы header и footer
- Сделал формы обратной связи с валидацией, при успешном отправлении появляется уведомление
- Добавил комментарии где нужно

### Какие маршруты и контроллеры вы бы добавили?

**Web маршруты:**
- `GET /` - главная страница
- `GET /service/{id}` - переход на детальную страницу
- `POST /application` - отправка заявки

**API маршруты:**
- `GET /api/services` - получение списка услуг
- `GET /api/services/{id}` - получение услуги по ID
- `POST /api/applications` - отправка заявки

**Контроллеры:**
- `ServiceController` - для веб-страниц
- `ApplicationController` - для обработки заявок

### Какие моменты задания показались вам не до конца определёнными?

1. **Форма заявки:**
   - Сохранять в БД?
   - Отображать в админке для отслеживания?
   - Письма админа и для клиента должны отправляться по email?
   - Интеграция с чем-то нужна, к примеру CRM?

2. **Связь заявки с услугой:**
   - Форма привязана к услуге или нет?

3. **Админ-панель:**
   - Какой функционал нужен в админке?
   - Нужна статистика или аналитика?
   - Нужен функционал для SEO?

4. **API:**
   - Нужно ли делать API?
   - Нужна версионирование?
   - Какие форматы ответов нужны?

5. **Масштабируемость:**
   - Ожидаемое количество услуг?
   - Нужна ли пагинация?
   - Ожидаемое количество заявок?

### Вопросы, которые я бы задал клиенту или тимлиду:

1. По функциональности:
    - Нужно ли сохранять заявки в базу данных или достаточно отправки на email?
    - Требуется ли уведомление администратора о новых заявках?
    - Нужна ли возможность ответа на заявку прямо из админ-панели?

2. По админ-панели:
    - Какие права доступа нужны? (суперадмин, модератор, редактор)
    - Нужна ли история изменений услуг (audit log)?
    - Требуется ли возможность массовых операций (удаление, изменение статуса)?

3. По API:
    - Планируется ли интеграция с внешними системами?
    - Нужна ли аутентификация через API токены?

4. По дизайну и UX:
    - Есть ли готовый дизайн-макет или нужно разработать с нуля?
    - Какие браузеры должны поддерживаться?
    - Нужна ли мобильная версия приложения?
